분해의 철칙 — 총량 보존

쪼개도 합은 그대로
총생산 = 양품(actual_qty) + 불량(defect_qty)
0104 — actual_qty는 양품이고, 총생산은 양품+불량.
불량비 dr = defect / 총생산 → 원료_불량 = 유량계총량 × dr
유량계 총량을 양품/불량 장수비로 안분 — 정상 + 불량 = 총량(이중차감 0).
g_res = ppg_good − g_main (≠ round(ppg×비율))
심재 PPG 잔여분은 잔여로 — 반올림 오차 누적을 막고 g_main + g_res = ppg_good 정확히.
filter_center_focus
모든 라인은 _add() 한 길목에서 소수점 2자리로 정규화됩니다 — 77.39999999 같은 부동소수점 잡음이 수불·eCount에 새지 않게. 정규화 후 0이 된 라인은 자동 제외.

5

결정과 트레이드오프 — 왜 이렇게 분해하나

카드를 눌러 채택 vs 기각
채택g_res = ppg_good − g_main (잔여).
기각round(ppg × 비율).
근거두 번 반올림하면 g_main+g_res ≠ ppg(오차 누적). 잔여는 합이 정확히 보존. ecount 정본 방식.
채택유량계 총량 × 불량비 dr로 양품/불량 안분.
기각양품분만 소모로 기록(불량 무시) / 별도 추정.
근거불량도 물리적으로 원료를 먹는다. 정상+불량 = 유량계 실측 총량(이중차감 0·보존). 불량 손실을 금액으로 잡으려면 분리가 필수.
채택defect_qty를 supplier_id 달린 PROD_OUTPUT 버킷으로 분리.
기각양품에 합산 / 아예 안 만듦.
근거재고장 VIEW가 supplier 달린 PROD_OUTPUT을 현재고에서 제외(#1). 불량은 추적되되 팔 재고 아님. 손실 금액은 원료/판재 라인에서만 → 이중계상 0.
채택모든 라인이 _add() 통과 시 round(2) + 0 라인 제외.
기각각 계산부에서 개별 반올림.
근거부동소수점 잡음(77.3999…)을 단일 길목에서 차단하면서도 잔여식 총량 보존을 유지. 분산 반올림은 누락·드리프트 위험.
채택소모량을 외부 주입하는 선택 파라미터.
기각분해 함수를 2벌로(수불용 / eCount용).
근거수불부=유량계 실측 그대로(0035), eCount=로스 포함 소모량 주입 — 같은 분해 로직, 입력만 다름(DRY). 심재 분할도 주입 총량 기준.
채택분해 로직을 bom.py로 추출·보존(SoT).
기각폐기 예정 ecount_etl_logic에 계속 의존.
근거모듈은 폐기돼도 BOM 규칙은 살아야 한다. 규칙만 단일 출처로 보존하고, spec과의 불일치(잔여·완전일치)를 코드 주석에 명시.
