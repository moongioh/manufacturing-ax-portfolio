3

화면이 안 늙는다 — 조용한 갱신 + 충돌 방지

plan 0055 — silent 폴링 · 상태전이 가드

현장은 생산계획을, 사무원은 문의 인박스를 하루 종일 띄워둡니다. 모든 화면이 "마운트 시 1회 fetch"라 상대 작업이 새로고침 전까진 안 보이고(stale), 오래된 화면으로 저장하면 상대 변경을 덮어써요(last-write-wins). 두 문제를 가볍게 막습니다.

1

30초 silent 폴링

공용 훅 usePolling 하나로 30초마다 기존 GET을 재조회(백엔드 변경 0). 부하는 3화면×5명×분당2회 ≈ 분당 30요청 — Cloud Run은 요청 처리시간만 과금이라 무시 가능(부수효과로 warm 유지).

2

"silent"의 3원칙

① 로딩 불간섭 — setLoading(true) 안 켬(30초마다 스피너 깜빡이면 그 자체가 사고). ② 조작 중 일시정지 — 모달 open·드래그 중·document.hidden이면 그 틱 스킵. ③ 실패 무시 — 폴링 실패는 토스트 0(CF 세션 만료 시 토스트 도배 방지).

＋

복귀 즉시 1회

visibilitychange로 탭을 다시 보는 순간 폴링 주기와 무관하게 즉시 1회 재조회 — "탭 다시 봤을 때 최신"을 보장.

3

상태전이 가드 = 409

상태머신이 있는 전이 엔드포인트는 기대 상태를 UPDATE … WHERE에 포함. 영향 행 0이면 409 반환 → 프론트는 "다른 사용자가 먼저 처리했습니다" 토스트 + 자동 재조회. 문의 확정·발주 취소/승인·생산계획 완료·출고 취소에 적용.

⚠

기존 도메인 409와 구분

출고 실재고 부족·생산일지 날짜 중복도 409라, 충돌 가드 409는 detail 메시지로 분기(예: [충돌] prefix). 안 그러면 "실재고 부족"이 "동시처리"로 둔갑.

bolt
웹소켓은 명시적으로 기각. 5인 규모에서 얻는 건 "30초→즉시"뿐인데, Cloud Run 상시 연결 과금 전환 + 스케일아웃 시 브로드캐스트 인프라(Redis/PubSub) + 재연결·재동기화 복잡도가 비대칭적으로 큽니다. 발주·문의에서 30초 지연은 업무가치 0 — 훅 하나로 충분.
4

비공개의 경계 — 나만 보는 메모

plan 0061 — 내용 수정 · 완전 비공개 개인 메모

collab_tasks는 태생이 부서 간 요청 구조(to_dept NOT NULL, 부서 스코프 권한). "수신 부서 없이 나만 보는 메모" 개념이 모델에 없었어요. 별도 테이블·별도 페이지를 새로 파지 않고 같은 테이블에 비공개를 1급 개념으로 추가합니다.

GET 필터 = (visibility='dept' AND [부서·보관 스코프]) OR (visibility='private' AND owner_email == 나)
private 분기는 부서 admin과 무관하게 owner_email == 현재 사용자에만 매칭 — 관리부도 타인 메모 못 봄. owner_email·to_dept=NULL은 서버가 stamp(클라 입력 불신).
shield_lock
완전 비공개 = 본인 외 누구도(관리부 admin 포함) 열람·수정·삭제 불가, 백엔드 강제. 슈퍼 어드민 전수 열람안은 폐기됐어요 — 직원 데이터 비공개 모니터링의 법적 리스크(개인정보보호법·근로기준법 고지)로 2026-06-15 사용자가 결정. 가장 큰 사고 클래스가 "메모 유출"이라, GET 한 곳 + 가드 한 곳으로 분기점을 좁히고 2계정 검증(V5/V6)으로 유출 0을 직접 관찰합니다.
edit
덤으로 고친 결함 둘. ① 제목·내용 수정 UI가 아예 없었어요 → 통합 PATCH /collab/tasks/{id} + 상세 모달 편집 모드. ② 칸반 카드 클릭이 죽어 있었다 — 칸반 뷰에 onTaskClick을 안 넘겨, 카드를 드래그만 되고 열 수가 없었죠. 클릭→모달 복구하되 드래그(grip 핸들)와 분리해 충돌 0. "받은 요청은 손도 못 댄다"는 사용성 사고를 닫았습니다.

6

폰과 PC, 둘 다 일급

plan 0065 — 반응형 단일 코드

협업 보드는 폰 홈화면 바로가기(#collab 딥링크)의 주 진입점이자 데스크톱에서 일이 굴러가는 보드 → 둘 다 first-class. 모바일 전용 빌드를 따로 만들지 않고, 반응형 단일 코드로 md: 미만에서만 분기합니다(백엔드·스키마 변경 0).

1

헤더 컴팩트

우측 버튼 클러스터 flex-wrap으로 오버플로 제거, 제목·통계 축소. 생성 액션은 우하단 FAB 기각 → 헤더 + 아이콘 버튼(하단이 기존 플로팅·네비바로 이미 붐벼서 폰 실검증 후 정정).

2

칸반 = 가로 스와이프

모바일은 컬럼을 scroll-snap 캐러셀(한 컬럼이 뷰포트 폭에 맞고 좌우 스와이프 + 인디케이터 점 ● ○ ○ ○). 데스크톱 가로 4컬럼 + grip DnD는 그대로.

3

모바일 상태변경 = 탭→모달

가로 스와이프(보기)와 터치 DnD가 충돌하니, 모바일은 카드 탭→상세 모달에서 상태 변경을 주 경로로. 드래그 한 방 이동은 포기하되 오작동 0.

4

캘린더 = 점 + 목록

모바일 월그리드는 텍스트 칩 대신 날짜별 점 마커(색=D-day 긴급도) + 날짜 탭→하단 선택일 목록. 데스크톱 7컬럼 풀 그리드 + 사이드 패널은 보존.

⚠

데스크톱 회귀 0이 1순위

모든 모바일 변경은 md:/lg:/sm: 분기로 격리 → 데스크톱 레이아웃은 종전과 동일해야 함(검증 V-D1/V-D2로 직접 차단). 재설계 아님.
