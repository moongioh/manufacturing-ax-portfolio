하나의 척추 — 사건을 쌓고, 숫자는 파생한다

도메인이 달라도 패턴은 같다
도메인무엇이 '사건'인가 (append-only)거기서 파생되는 숫자
재고이동 ±qty (입고·생산·출고·샘플·실사)현재고 = SUM(qty) · 가용 = total − reserved

자금청구·입금·지급 사건미수금(AR)·미지급금(AP) 잔액 · 자금일보 시재
불량불량 거래처 귀속 사건품목·월별 귀속 손실 (현재고에선 제외)

그래서 제조 ERP엔 "현재값 컬럼"이 거의 없습니다. "지금 이 숫자가 왜 이렇지?"는 항상 이력으로 설명되고, 되돌리기는 행 삭제가 아니라 반대부호 사건으로 합니다(감사 가능·회계 역분개와 동형). 이 한 패턴을 이해하면 9개 워크스루가 전부 같은 이야기의 변주로 읽힙니다.

5

결정과 트레이드오프 — 시스템을 왜 이렇게 지었나

카드를 눌러 채택 vs 기각을 펼쳐보세요
채택이벤트 소싱 — append-only 사건 + 파생 읽기.
기각각 도메인이 '현재값' 컬럼을 직접 갱신.
근거ERP 최악의 사고는 조용한 데이터 오염. 현재값 갱신은 동시성·중복차감에 취약하고 "왜 이 숫자?"를 설명 못 한다. 사건은 전부 감사 가능하고 재계산으로 복원된다. 비용=파생 SUM(현 규모 무시 가능).
채택단일 PostgreSQL 유지(벡터는 pgvector).
기각NoSQL 이전 · 별도 벡터DB · RAG 인프라.
근거데이터 규모가 작고 관계·트랜잭션 무결성이 핵심. 분산 저장은 정합성 비용만 늘린다. 임베딩도 데이터량이 작아 별도 인프라가 과설계(RAG 구상 폐기). 하나의 진실 저장소가 운영을 단순하게.
채택FastAPI + React + Cloud Run 컨테이너.
기각tkinter 데스크톱 + PyInstaller .exe 배포.
근거.exe는 PC마다 설치·버전 드리프트·원격 협업 불가. 웹은 한 번 배포로 전원 최신, 모바일 열람, 무중단 업데이트. 트레이드오프=인프라 학습비용(컨테이너·CI/CD)이나 1회성.
채택Cloud Run + Cloudflare Access(Zero Trust 무료).
기각GCP HTTPS 로드밸런서(상시 가동).
근거2인 사용에 상시 LB는 idle만으로 월 $30+. $0 idle 원칙 — 안 쓰면 $0. Cloudflare가 SSO 게이트 + TLS를 무료로. 단 per-call AI(Gemini API)는 호출 시에만 과금이라 허용(상시 인프라가 아님).
채택공통 프리미티브 + 대형 모듈 수술(touched-시점).
기각전면 재작성 / 일괄 리팩토링 스윕.
근거최우선 규칙 = "돌아가는 기능을 안 깨는 게 새 기능보다 우선". 빅뱅은 회귀 폭탄. 건드릴 때만 ~300줄 넘는 파일을 hooks/components로 쪼갠다. 트레이드오프=정리 속도↓이나 안전.
채택승인된 plan → 구현 · pre-commit 캡 · 드리프트 검증단.
기각산문 규율만(에이전트 자율 신뢰).
근거AI 에이전트는 규율을 잊는다 — 산문은 드리프트한다. 기계가 강제(STATE 16KB 캡·INDEX 자동생성·세션 역할 쓰기소유)해야 멀티세션에서 원장 경합·이중기록이 안 난다. 트레이드오프=절차 오버헤드이나 인수 안전성↑.
