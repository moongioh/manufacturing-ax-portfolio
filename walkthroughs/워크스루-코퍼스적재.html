결정과 트레이드오프 — 왜 이렇게 설계했나

카드를 눌러 채택 vs 기각을 펼쳐보세요
채택앱 serving SSoT는 Postgres ontology.*에 보관. GCS에는 백업만.
기각지식그래프를 GCS에 JSON/RDF 형태로 직접 보관.
근거GCS는 그래프 탐색(traversal) 쿼리가 불가능하여 앱에서 직접 읽어 쓰기에 적합하지 않습니다. Postgres 2테이블 구조가 serving SSoT로 작동합니다.
채택Vertex AI / Azure 등 SaaS 임베딩 API (종량제 per-call).
기각상시 기동형 Vertex AI Vector Search 엔드포인트 구축.
근거상시 켜져 있는 벡터 검색 엔드포인트는 $0 idle 원칙을 위반하여 유휴 상태에서도 고정 비용을 발생시킵니다. 호출당 과금되는 종량제 API가 적합합니다.
채택의미 기반으로 후보를 제안하고, 사람이 최종 confirm하여 트리플로 승격.
기각100GB 코퍼스에서 지식그래프를 자동 추출하여 즉시 적재.
근거완전 자동 추출은 환각(Hallucination)과 비정제 데이터 유입으로 정본 그래프를 심각하게 훼손합니다. 사람이 확인하는 큐레이션 게이트(Human-in-the-loop)를 유지합니다.
