제조 중소기업 웹 ERP 1인 구축

18인 제조·화학 중소기업의 웹 ERP를 1인으로 설계·구축해 운영한 프로젝트(재직 2026.04~2026.07) — 입사 4주 차에 첫 실가동을 시작해 약 2개월에 걸쳐 주요 업무 도메인으로 확장했습니다. 자체 소프트웨어가 없고 데이터가 6~7곳에 흩어져 있던 환경에서, 데이터를 단일 정합 구조로 옮기고 그 위에 웹 ERP를 만들었습니다. 비정형 문서 16GB는 시멘틱 문서허브로 검색 가능하게 만들었고, 개발에 사용한 AI 에이전트 작업 규율은 오픈소스 harness-scope로 공개했습니다.

In English: Solo design, build and operation of a cloud full-stack ERP (FastAPI · React · PostgreSQL · Cloud Run) for an 18-person manufacturing/chemical SME in Korea — first production use four weeks in, expanded over about two months. Unified 6–7 scattered sources of truth into a single event-sourced data backbone with domain-ontology validation, added a semantic document hub over 16GB of unstructured files, and open-sourced the AI-agent workflow tooling used to build it as harness-scope.

오픈소스 — harness-scope

설치해서 직접 확인할 수 있는 공개 산출물

위 ERP 개발에 사용한 AI 에이전트 작업 규율을 관측 도구로 만들어 Apache-2.0 오픈소스로 공개했습니다. 에이전트 세션 로그(JSONL)를 파싱해 선언적 룰(R1~R13)로 턴 단위 판정하고, Session Replay · Rule Health · 토큰 손실 핀포인트 뷰어와 MCP 서버(10툴)를 제공합니다. 검증 하니스 16종 · 3-OS CI로 상시 검증됩니다.

시멘틱 문서허브 — 비정형 16GB를 검색 가능하게

정형(온톨로지) + 비정형(문서허브) 검색

ERP가 정형 데이터를 닫았다면, 문서허브는 비정형을 닫은 두 번째 축입니다 — 폴더와 개인 PC에 흩어져 검색조차 어렵던 수년치 문서 16GB를 정제(2,809문서)해 pgvector 임베딩(13,759청크·적재 에러 0)으로 올리고, 키워드+벡터 RRF 융합에 온톨로지 개념 채널을 더한 하이브리드 검색, "최신 정본이 어느 파일인가"를 DB UNIQUE 제약으로 강제하는 정본 레지스트리, d3-force 의미지도, 근거 문서를 인용하는 RAG 초안 생성까지 하나의 파이프라인으로 만들었습니다.
